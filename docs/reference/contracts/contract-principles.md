# Cross-language contract principles

Language-agnostic patterns distilled from the two generation bridges in
`legal-ai`. Both are one-directional codegen pipelines; the principles below
are what makes them safe.

## 1. One owner per contract; the other side is generated

Every cross-language boundary has a **single source of truth in one language**,
and the other language's types are generated artifacts — never hand-authored,
never hand-edited.

- Data model contract → **TypeScript Zod owns it** (`schemas.ts`); Python
  Pydantic is generated.
- Service API contract → **Python FastAPI owns it** (OpenAPI at
  `/openapi.json`); the TypeScript SDK is generated.

The ownership follows where the logic lives: the domain model is authored in
the TS knowledge-graph package, so it owns the schema; the services are
implemented in Python, so they own their API surface.

## 2. Generation direction follows the runtime dependency

The consumer generates a client/types *from* the producer's contract, in the
producer's native emission format:

- Zod → JSON Schema (Draft 2020-12) → Pydantic.
- FastAPI → OpenAPI → TypeScript fetch client.

Pick the interchange format the owning framework emits natively (JSON Schema
for Zod v4 via `z.toJSONSchema`, OpenAPI for FastAPI) rather than inventing a
bespoke one.

## 3. Generated code is committed, but marked and quarantined

Generated files are checked in (so consumers build without running codegen) yet
clearly fenced off:

- Header banners: "Auto-generated by @hey-api/openapi-ts",
  "Auto-generated — do not edit by hand".
- Physical separation: `_generated.py` vs `_augment.py`; `*.gen.ts` under
  `src/sdk/`.
- A **stable public surface** re-exports from the generated layer so callers
  never import the volatile file directly. Python's `schemas/__init__.py`
  chooses augmented-vs-generated provenance per name; the SDK's `index.ts`
  re-exports `types.gen`/`sdk.gen`.

## 4. Determinism is a first-class requirement

Generated output must be byte-reproducible so drift can be detected by
re-running the generator and diffing:

- Sort keys recursively and emit a trailing newline (JSON Schema emit).
- `--disable-timestamp` on `datamodel-code-generator`.
- Generator versions are pinned (`@hey-api/openapi-ts@0.82.3`).

Codegen options are also tuned for *downstream ergonomics*, not just
correctness — e.g. `reused: "inline"` and the `z.date()` override exist purely
so the generated Pydantic reads naturally (`node.id`, `datetime`).

## 5. Drift prevention = regenerate-and-diff, not "trust the committer"

The canonical guard (`check-types-in-sync.sh`) snapshots output hashes,
re-runs the pipeline, and fails if anything changed. It hashes *file content*
rather than consulting `git status`, so it catches drift regardless of
tracked/staged state — it tests generation determinism against what is on disk.
The failure message names the fix ("Run `pnpm generate:python-types` and commit
the diff").

**Precondition for this guard:** the pipeline must be reproducible in a
stateless checkout. It works for Zod→Pydantic (pure file-in, file-out). It does
**not** work for OpenAPI→SDK, whose generator reads a *live service* URL — so
that bridge has no regenerate-and-diff gate and relies on downstream type errors
instead. Principle: **a contract you can't regenerate offline, you can't
drift-check offline.**

## 6. Name the gaps; work around them on the generated side, not the source

Interchange formats lose information (JSON Schema can't express cross-field
predicates, `Omit<>`, tuples; OpenAPI SSE codegen hit a runtime bug). The rule
observed everywhere: **do not distort the source of truth to satisfy the
generator.** Instead:

- Restore lost semantics in the augmentation layer (`_augment.py` subclasses
  with `@model_validator`).
- Re-apply necessary fixes deterministically in a post-generation patch
  (`patch-sse-streaming.ts` — idempotent and pattern-guarded so it fails loudly
  if the generated shape changes).

Each known gap is documented with its deliberate workaround, so the lossiness
is visible rather than silently accepted.
